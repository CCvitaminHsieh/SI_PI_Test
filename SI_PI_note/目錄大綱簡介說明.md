這個章節的目的不在於做高難度模型，而是要建立 "HFSS的語言" 與 "微波工程的基本"觀念 ，以及為什麼我們會需要 EM 模擬，
再來是 HFSS 的基本操作流程 (建模->設定->仿真->結果)，以上為封裝互連 (TSV, bump, RDL) 的基礎。

1. 信號完整性的基本物理是什麼
	- 為什麼高速訊號不是 "電壓" 而是 "電磁波"
		- **當訊號的上升時間變快、頻率變高時，電路不再是低頻的 lumped 元件 (RLCG)，而變成分布式電磁場系統**。電壓只是電磁波在導體上的「邊界條件」，不是能量本體。
		- 電路尺寸 ≪ 訊號波長，電壓/電流在整個元件上幾乎同時變化
		- 低頻（例如 1 kHz）時：
			- 導線長度 ≪ 訊號波長
			- 整條線的電壓幾乎同時變化
			- 可以用 lumped model（RLC）描述
			
			所以你會覺得「電壓在傳」。
			
			但高速訊號（例如 1 Gbps）：
			
			- 上升時間 100 ps → 有效頻率 ~3–5 GHz
			- 波長在 FR4 中只有 3–5 cm
			- PCB trace 長度常常是 5–10 cm
			
			此時：
			
			- **線路長度 ≈ 波長**
			- 電壓不再同時變化
			- 必須用 Maxwell 方程式 + 傳輸線模型
			
			這時訊號行為完全是**電磁波傳播**。
	- 
- 為什麼 trace 是傳輸線
	- Trace 之所以必須被視為 傳輸線（Transmission Line），核心原因是：當訊號的上升時間足夠快、頻率足夠高時，電壓與電流不再能在整條線上「同時」變化，電磁場沿著線路以有限速度傳播，線路行為必須用傳輸線方程式描述，而不能再用 lumped RLC 模型。這不是 PCB 特有，而是所有導體在高速下都會變成傳輸線。
	- **能量以電磁波形式沿介質傳播，而不是電壓在銅裡跑**

		高速訊號的能量由 Poynting vector 描述：
		
		\vec {S}=\vec {E}\times \vec {H}
		
		能量沿著 trace 旁的介質（FR4、空氣）前進，而不是在銅裡。
		
		這意味著：
		
		- Trace 是導引電磁波的「波導」
		- 電壓只是電磁波在導體上的邊界條件
		- 電流是磁場的結果
		
		既然能量以波的形式傳播，trace 就自然是傳輸線。
		電磁波沿金屬表面傳播：能量主要在介質，不在金屬
		高速訊號的能量由 Poynting vector 描述：
		\vec {S}=\vec {E}\times \vec {H}
		這個向量指向 沿著 trace 前進的方向，而能量主要集中在：
		- 介質（FR4、空氣）中
		- 金屬表面附近的極薄區域（skin depth）
		金屬本身並不「傳遞」能量，它只是提供邊界條件讓電磁波被導引。
		
		⚡ 電容儲存電場、電感儲存磁場 → 電磁波就是交替的 E/H 場
		
		你說的這句話非常關鍵：
		
		電容儲存或釋放電場，電感儲存或釋放磁場
		
		電磁波本質上就是：
		
		- 電場（E）變化 → 產生磁場（H）
		- 磁場（H）變化 → 產生電場（E）
		
		這個交替過程沿著 trace 前進，就是訊號傳播。
	🧲 為什麼金屬會發熱？（關鍵：不是完美導體）

金屬不是完美導體，因此：

- 電磁波的磁場會在金屬表面誘發電流（表面電流）
- 這些電流因為金屬有電阻 → 產生 I²R 損耗
- 這些損耗就是 **導體損耗（conductor loss）**

這就是 trace 會發熱的原因。

這個現象的三個核心點

- **電磁波的一小部分能量滲入金屬（skin depth）**
- **在金屬表面誘發電流**
- **電阻造成能量耗散 → 熱**

這完全符合你說的「電磁波一部分能量逸散至金屬」。

🧪 Skin depth：為什麼能量只滲入金屬表面？

電磁波在導體內會被快速衰減，其衰減深度稱為 skin depth：

\delta =\sqrt{\frac{2}{\omega \mu \sigma }}

例如在 1 GHz：

- 銅的 skin depth 約 **2 µm**
- PCB trace 厚度通常 18–35 µm

→ **只有最外層 2 µm 有電流，其他部分幾乎沒有能量進入**

這就是為什麼：

- 電流集中在表面
- 表面粗糙度會增加損耗
- 高頻 trace 會更熱

🔥 兩種損耗：導體損耗 + 介質損耗

電磁波沿 trace 傳播時，能量會因兩種機制被消耗：

1. **導體損耗（R′）**

- 來自金屬表面電阻
- 與 skin effect、表面粗糙度有關
- 造成熱

2. **介質損耗（G′）**

- 來自 FR4 的介電損耗（tanδ）
- 電場在介質中振盪時能量被吸收
- 也造成熱

既然高頻電流只在 **skin depth（約 2 µm）** 的表面流動，那為什麼 PCB trace 還要做 **18–35 µm** 這麼厚？  
是不是只要做 2 µm 厚就夠了？  
是不是怕太薄會造成反射？

答案是：**不是為了反射，而是為了「直流與低頻電流承載能力、可靠性、製程限制、機械強度、電鍍均勻性」等多重原因。**

下面把整個機制完整拆開，你會看到 PCB 厚度與電磁波傳播其實是兩個不同層次的問題。

---

## 🧲 1. 高頻電流只在表面流動，但 PCB trace 不能只做 2 µm

高頻電流確實只在 skin depth 內流動：

[ \delta = \sqrt{\frac{2}{\omega \mu \sigma}} ]

1 GHz 時銅的 skin depth 約 **2 µm**。  
但 PCB trace 厚度常見：

- 0.5 oz → 約 18 µm
- 1 oz → 約 35 µm
- 2 oz → 約 70 µm

這些厚度遠大於 skin depth。

但這不是浪費，而是因為：

### **PCB trace 的厚度主要是為了「直流與低頻電流」的承載能力**

- 電源線、地線、電源分配網路（PDN）
- 甚至高速線在 idle 時也有 DC bias

DC 電流不會只走表面，而是整個截面都會導電。  
如果只做 2 µm 厚，DC 電阻會高到不實用。

---

## 🧱 2. PCB 製程本身無法做「2 µm 厚的銅線」

PCB 的銅厚不是任意選的，而是製程決定的：

- 銅箔原料本身就是 12–35 µm
- 電鍍、蝕刻都需要一定厚度才能均勻
- 太薄會直接被蝕刻穿
- 太薄會造成線寬不均、開路、可靠性問題

所以 **2 µm 厚的 trace 在 PCB 製程上根本做不出來，也不可靠**。

---

## 🔥 3. 厚度不是為了避免反射，反射與「阻抗」有關，而不是厚度

反射的來源是：

[ Z_{\text{trace}} \neq Z_{\text{load}} ]

而 trace 的 characteristic impedance 主要由：

- 線寬
- 高度（trace 到 reference plane 的距離）
- 介電常數
- 線形（microstrip/stripline）

決定。

**厚度只會稍微影響阻抗（因為影響 effective width），但不是反射的主因。**

所以：

- 厚度不是為了避免反射
- 厚度不是為了讓電磁波「不會斷掉」
- 厚度不是為了讓電磁波「更好傳」

厚度主要是 **電流承載能力 + 製程限制 + 機械強度**。

---

## 🧲 4. 電磁波能量主要在介質，不在銅裡

這是你前面已經掌握的核心概念：

- 能量沿著 trace 外的介質（FR4、空氣）傳播
- 銅只是提供邊界條件
- 只有極薄的 skin depth 會有電流

所以：

> **銅厚度對電磁波傳播影響很小，對 DC/低頻電流影響很大。**

---

## 🧪 5. 為什麼厚度越厚，高頻損耗反而不一定變好？

因為：

- 高頻電流只在最外層 2 µm
- 但銅越厚，表面粗糙度（copper roughness）通常越大
- 粗糙度會增加高頻損耗（Huray model）

所以高速 PCB 常用：

- **低粗糙度銅（VLP、HVLP）**
- 而不是盲目加厚銅

這也再次證明厚度不是為了高速訊號。

---

## 🧩 6. 一句話總結

> PCB trace 之所以做 18–35 µm 厚，不是因為電磁波需要這麼厚，而是因為 DC/低頻電流承載能力、製程限制、機械強度與可靠性需求。高速電磁波只在表面 2 µm 內傳播，與厚度無關，也不會因為太薄而反射。

📡 為什麼這些現象證明 trace 是傳輸線？

因為：

- 能量以電磁波形式沿著 trace 傳播
- 電場與磁場沿著線路分布
- 金屬與介質都會吸收能量
- 必須用 R′L′C′G′（每單位長度）描述損耗與傳播

這些都是典型的 **傳輸線行為**，不是 lumped RLC 能描述的。

電磁波沿 trace 表面傳播時，因金屬不是完美導體，部分能量會滲入金屬表面形成表面電流，並因金屬電阻而轉成熱，這就是導體損耗與 trace 發熱的來源。

是的，即便在同一層（同一個 copper layer），**實際上也常常會被分成數個不同的功能區域：電源區、訊號區、地區**。這不是因為電磁波會「斷掉」或「反射」，而是因為 **PCB 製程、佈線需求、電流承載能力、阻抗控制與回流路徑** 等因素共同決定的。

下面把這件事拆成幾個層次，你會更清楚為什麼同一層會有不同區域，以及這些區域對高速訊號的影響。

---

## 🧭 同一層為什麼會分成「電源、訊號、地」？

在 PCB 設計中，每一層的用途不是固定的，而是依照需求分配。常見原因包括：

### 1. **電流承載能力不同**

- 電源線需要較寬、較厚的銅來承載大電流
- 訊號線需要細線寬來控制阻抗
- 地區域通常鋪滿銅以降低阻抗、提供回流路徑

所以同一層自然會出現不同用途的區域。

---

### 2. **佈線密度與 routing 需求**

- 高速訊號需要走特定路徑（等長、差分、避開噪聲源）
- 電源線通常較短、較粗
- 地區域需要盡量連續

因此 layout 工程師會在同一層上切出不同區域。

---

### 3. **製程與成本限制**

- 多加一層 PCB 成本會大幅增加
- 設計者會盡量在有限層數中完成所有功能

所以同一層常常混合不同用途。

---

## ⚡ 那這樣會不會影響高速訊號？（關鍵在「回流路徑」）

高速訊號最在意的不是「同層是否混合」，而是：

> **高速訊號是否有連續、低阻抗的回流路徑（return path）？**

如果同一層的地區域被切割、破洞、分區，就會造成：

- 回流路徑繞遠
- 迴路面積變大
- EMI 增加
- 阻抗不連續
- 反射、串擾、輻射問題

這是高速 PCB 設計最常見的問題之一。

---

## 🧲 電磁波傳播與「同層分區」的關係

你前面已經掌握了核心概念：

- 訊號能量在介質中走
- 回流電流沿著 reference plane 下方走
- 金屬只是邊界條件

因此：

### **只要 reference plane（通常是地）連續，訊號就能穩定傳播。**

反之：

### **如果地被切割、分區、開槽，電磁波的場分布會被破壞，造成嚴重問題。**

所以同層分成「電源、訊號、地」不是問題，  
問題是：

- 地是否連續？
- 電源是否提供良好 return path？
- 訊號是否跨越 plane split？

---

## 🧩 那厚度與反射有關嗎？

你前面問的：

> 鋪太薄會不會導致電磁波反射？

答案是 **不會**。

反射來自阻抗不連續：

[ Z_{\text{trace}} \neq Z_{\text{load}} ]

而阻抗主要由：

- 線寬
- 介質厚度
- 介電常數
- 與 reference plane 的距離

決定。

銅厚度只會微小影響 effective width，不是反射主因。

---

## 🧠 一句話總結

> 同一層可以同時存在電源、訊號、地區域，這是正常的 PCB 設計方式。高速訊號真正需要的是「連續的回流路徑」，而不是「整層都要是地」。銅厚度與反射無關，反射來自阻抗不連續。
電磁波沿著 PCB trace 傳遞時，如果途中遇到 **破洞（plane split）、缺口、開槽、銅皮中斷、return path 不連續**，它的行為不是「繞過去」那麼簡單，而是會出現一系列可預期、可量化的電磁場現象。這些現象是高速 PCB 設計中最危險、最容易被忽略的問題之一。

下面用你已經掌握的「電磁波沿金屬表面、在介質中傳遞」的觀念，完整解釋電磁波遇到破洞時會怎麼跑。

## 🌊 電磁波遇到破洞時的核心行為（最重要的三件事）

1. **回流路徑被切斷 → 電磁波無法維持原本的場分布**
2. **能量被迫繞路 → 迴路面積變大 → 輻射增加（EMI）**
3. **阻抗不連續 → 反射、損耗、抖動、眼圖閉合**

這三件事同時發生，而且是物理必然，不是 layout 技巧能「補救」的。

---

## 🧲 電磁波遇到破洞時的場分布變化

高速訊號的能量主要在：

- trace 與 reference plane（通常是 GND）之間的介質中
- trace 下方的 return current 形成磁場閉合路徑

當 reference plane 有破洞時：

### 1. **回流電流無法走最短路徑**

原本的 return current 緊貼在 trace 下方（磁場最小能量路徑）。  
但破洞讓它無法走原本的路，只能：

- 繞過破洞邊緣
- 跳到其他 plane
- 經由電容耦合到遠處的地
- 經由 stitching via 重新回到地

這會造成迴路面積急劇變大。

---

### 2. **迴路面積變大 → 輻射（EMI）急劇上升**

迴路面積越大，磁場越強，輻射越大：

[ \text{EMI} \propto \text{迴路面積} ]

這就是為什麼高速訊號跨 plane split 會造成：

- EMI fail
- 眼圖閉合
- jitter 增加
- 反射與損耗上升

---

### 3. **阻抗不連續 → 反射**

傳輸線阻抗由 L′/C′ 決定，而 L′、C′ 由場分布決定。

破洞會讓：

- 電場找不到 reference plane → C′ 下降
- 磁場迴路變大 → L′ 上升

因此 characteristic impedance 會突然變大：

[ Z_0 = \sqrt{\frac{L'}{C'}} ]

這就是反射的來源。

---

## 📡 電磁波實際「怎麼跑」？

可以把它想成三段：

### **(1) 電磁波到達破洞邊緣 → 場分布被迫改變**

電場找不到地面 → 向旁邊擴散  
磁場找不到回流 → 向外擴張

### **(2) 電磁波沿著破洞邊緣「滑行」**

這是因為：

- return current 會沿著破洞邊緣走最短路徑
- 電磁波的能量會跟著 return path 走

這會造成局部場強增加 → EMI 熱點。

### **(3) 電磁波跳到下一個可用的 reference plane**

靠的是：

- 平行平面間的電容耦合
- stitching via
- 電源/地平面間的分布式電容

如果沒有 stitching via，能量會跳得更遠 → EMI 更大。

---

## 🔥 最糟糕的情況：訊號跨越 plane split

這是高速 PCB 設計的禁忌。

訊號跨越 split 時：

- return current 無法跟著走
- 迴路面積變成原本的 10–100 倍
- EMI 可能瞬間超標
- 反射造成 eye diagram collapse
- 低頻時也會有噪聲（ground bounce）

這不是「有點不好」，而是「完全不能接受」。

---

## 🧠 一句話總結

> 當電磁波沿著 trace 傳遞時遇到破洞，回流路徑被切斷，能量被迫繞路，造成迴路面積變大、阻抗不連續、反射與 EMI 急劇上升。電磁波不會「穿過破洞」，而是沿著破洞邊緣跑，並跳到下一個可用的 reference plane。

---

在「完美導體」的假設下，電場與磁場在金屬邊界的行為會變得非常極端、非常乾淨。你問的核心是：

> 完美導體不會因電場滲入而發熱，那磁場會不會造成能量損耗？

關鍵答案是：

> **在完美導體中，電場與磁場都無法進入導體內部，因此不會有能量滲入金屬，也不會有任何形式的發熱。**

## 🧲 完美導體的三個核心條件

完美導體（perfect conductor）定義上滿足：

1. **電阻 = 0**
2. **電場在導體內部 = 0**
3. **磁場在導體內部 = 0（在穩態 AC 下）**

這三件事一起成立，導致：

- 電磁波無法進入導體
- 所有能量都沿著導體表面傳遞
- 導體內部沒有電流、沒有損耗、沒有熱

---

## ⚡ 電場為什麼不能進入完美導體？

因為：

- 導體內自由電子會立即重新分布
- 任何試圖進入的電場都會被完全抵銷
- 導體內部的 E = 0

這是靜電學的基本邊界條件。

---

## 🧲 那磁場呢？磁場不是可以穿透金屬嗎？

這是你問題的重點。

在「真實導體」中：

- 磁場可以進入金屬一小段距離（skin depth）
- 這會誘發表面電流
- 電阻造成 I²R → 發熱

但在「完美導體」中：

> **磁場也無法進入導體內部。**

原因是：

- 完美導體的電阻 = 0
- 任何磁場變化都會誘發「無損耗的電流」
- 這些電流會產生反向磁場
- 完全抵銷外部磁場的穿透

這與超導體的「完全抗磁性（Meissner effect）」非常類似。

---

## 🔥 所以完美導體不會發熱，因為：

### 1. **沒有電場滲入 → 沒有電阻電流 → 沒有損耗**

### 2. **沒有磁場滲入 → 沒有渦電流損耗 → 沒有損耗**

### 3. **表面電流無電阻 → 不會產生 I²R 熱**

因此：

> **完美導體不會因電場或磁場而發熱。**

---

## 📡 那電磁波怎麼沿著完美導體傳播？

電磁波仍然沿著導體表面傳播，但：

- 能量完全在介質中
- 導體只提供邊界條件
- 表面電流存在，但沒有損耗

這是「理想傳輸線」的極限情況：

- R′ = 0
- G′ = 0
- 只有 L′、C′
- 無損耗傳播

這就是理想 50 Ω 傳輸線的數學模型。

---

## 🧠 一句話總結

> 在完美導體中，電場與磁場都無法進入導體內部，因此不會有能量滲入金屬，也不會產生任何形式的發熱。磁場也不會造成損耗，因為它會被表面無損耗電流完全排斥。

## 🌐 為什麼互連需要用 S 參數？（核心原因）
因為高速互連不是低頻電路，而是**電磁波傳播系統**。

---

## ⚡ Maxwell → 傳輸線 → S 參數：三層關係
### 1. Maxwell 方程式  
描述電磁場如何在介質中傳播。

### 2. 傳輸線方程式  
由 Maxwell 推導而來，描述波在 trace 中的行為：

- characteristic impedance  
- propagation constant  
- 反射、損耗、相位  

### 3. S 參數  
把「波」的行為量化成：

- **S11：反射多少？**  
- **S21：傳輸多少？**  
- **S12：反向串擾多少？**  
- **S22：另一端的反射？**

這三者是連續的物理層級，而 S 參數是工程師能直接量測與使用的那一層。

## 📡 為什麼不能用 RLC 模型描述高速互連？
因為高速互連的行為不是集中元件，而是**分布式電磁場**：

- trace 的 L′、C′ 是每單位長度的  
- 電場與磁場沿著線路分布  
- 反射來自阻抗不連續  
- 損耗來自導體損耗與介質損耗  
- 相位延遲來自介電常數  

這些現象都不是 RLC 能描述的，但 S 參數可以。

## 🧲 S11：為什麼重要？
S11 告訴你：

- 阻抗不連續有多嚴重  
- via、connector、pad、stub 造成多少反射  
- 眼圖會不會閉合  
- DDR/PCIe/USB 是否能鎖定  

如果 S11 很差，訊號會被反射回去，造成：

- jitter  
- timing error  
- eye collapse  
- bit error  

這是高速互連最關鍵的指標之一。

## 🚀 S21：為什麼重要？
S21 告訴你：

- 訊號能量有多少成功傳到接收端  
- trace 的 insertion loss  
- 介質損耗、導體損耗、表面粗糙度的影響  
- 頻寬是否足夠  

高速介面（PCIe、USB3、HDMI）都有嚴格的 S21 頻寬要求。

核心概念是：

> **S21 會下降，是因為電磁波沿著傳輸線前進時，部分能量被導體吸收（I²R 熱）與介質吸收（tan δ），因此到達接收端的能量變少。**

下面把整個物理機制拆成你已經熟悉的「電磁波沿導體表面傳播」的語言來說明。

---

## 🌐 電磁波沿 PCB trace 傳遞時，能量主要在介質，但會「滲入」金屬表面

你前面已經掌握：

- 高頻電流只在 **skin depth**（例如 1 GHz 約 2 µm）內流動
- 電磁波能量主要在 **介質（FR4）** 中
- 金屬只是邊界條件，但不是完美導體

因此：

- 電磁波的磁場會在金屬表面誘發電流
- 這些電流因金屬有電阻 → 產生 I²R 熱
- 這部分能量被「吃掉」了，不會再傳到接收端

這就是 **導體損耗（conductor loss）**。

---

## ⚡ S21 如何反映這些能量損失？

S21 定義為：

[ S_{21} = \frac{\text{到達接收端的波}}{\text{送入的波}} ]

如果導體損耗存在：

- 一部分能量變成熱
- 一部分能量被介質吸收
- 一部分能量被反射（S11）
- 剩下的才是 S21

因此：

> **導體損耗越大，S21 越低（更負）。**

在 dB 中：

[ S_{21}(dB) = 20 \log_{10} |S_{21}| ]

例如：

- −1 dB → 約 89% 能量到達
- −3 dB → 50% 能量到達
- −10 dB → 10% 能量到達

這些損耗就是 trace 變熱的來源。

---

## 🧲 為什麼導體損耗會隨頻率上升？

因為 skin effect：

[ \delta = \sqrt{\frac{2}{\omega \mu \sigma}} ]

頻率越高：

- skin depth 越薄
- 電流集中在更薄的表面
- 有效電阻上升
- I²R 損耗上升
- S21 下降更快

這就是為什麼高速 PCB trace 的 S21 曲線會隨頻率快速下滑。

---

## 📡 S21 不只反映導體損耗，也反映介質損耗

介質損耗（tan δ）會讓電場能量被介質吸收：

- 造成熱
- 造成 S21 下降
- 造成相位延遲（群延遲變化）

因此 S21 包含：

- **導體損耗（R′）**
- **介質損耗（G′）**
- **反射造成的能量損失（S11）**

這些全部來自 Maxwell 方程式的場分布。

---

## 🧠 一句話總結

S21 會下降，是因為電磁波沿著導體表面傳遞時，部分能量因金屬電阻（I²R 熱）與介質損耗被吸收，導致到達接收端的能量變少；因此 S21 是量化「能量損失」最直接的指標。
是的，**介質損耗會把電磁波的一部分能量轉換成熱能**，而且這個機制和你前面理解的「偶極矩偏轉」完全連在一起，只是需要把物理圖像補完整，才能真正看清楚能量去哪裡了。

下面用你已經熟悉的「電磁波沿著 PCB 介質傳遞」的觀念，把介質損耗的本質拆成清楚的三層。

---

## 🌐 介質損耗為什麼會把電磁波能量變成熱？

介質損耗（dielectric loss）來自介質內部的**極化延遲**。  
當電磁波的電場 (E(t)) 通過介質時：

- 電子雲被拉伸
- 離子微小位移
- 偶極子旋轉對齊

但這些極化反應**不是瞬間完成的**，因為分子有慣性、摩擦、束縛力。

結果是：

- 偶極矩的反應會**落後**外加電場
- 這個落後的部分（out-of-phase component）會消耗能量
- 消耗的能量轉成**熱**

這就是介質損耗的本質。

---

## ⚡ 介質損耗的數學表現：tan δ

介質損耗用損耗角正切表示：

[ \tan\delta = \frac{\text{損耗能量}}{\text{儲存能量}} ]

tan δ 越大：

- 介質吸收越多能量
- 電磁波衰減越快
- S21 下降越快
- PCB 越容易發熱

FR4 的 tan δ 大約 0.015–0.02，所以在 GHz 領域損耗很明顯。

---

## 🔥 介質損耗與導體損耗的差別

兩者都會讓 S21 下降，但機制不同：

- **導體損耗（R′）**：電磁波在金屬表面誘發電流 → I²R 熱
- **介質損耗（G′）**：介質極化延遲 → 分子摩擦 → 熱

兩者都會讓能量變成熱，但位置不同：

- 導體損耗 → 銅變熱
- 介質損耗 → FR4 變熱

在高速 PCB 中，**介質損耗通常比導體損耗更嚴重**，尤其在 5–20 GHz。

---

## 📡 電磁波能量如何「被介質吃掉」？

可以把它想成：

1. 電場推動介質中的偶極子
2. 偶極子因為阻尼（摩擦）無法完全跟上
3. 這個「落後」的部分需要能量
4. 這些能量來自電磁波
5. 能量被分子運動吸收 → 變成熱

所以：

> **介質損耗不是抵抗電磁波，而是吸收電磁波。**

這和你前面問的「偶極矩偏轉會不會抵抗電磁波」剛好呼應：  
它不是抵抗，而是吸收 → 變熱 → S21 下降。

---

## 🧠 一句話總結

介質損耗會讓電磁波的一部分能量被介質吸收，轉換成熱能。這是因為介質內的偶極子、電子雲與離子在跟隨電場變化時有延遲與阻尼，這些「落後的極化」會消耗能量，造成 S21 下降與 PCB 發熱。

# HFSS 的完整 workflow（從零開始建一個可用的 EM 模型）

## 1. 新建 HFSS 工程：Project / Design 的概念

HFSS 的層級分成兩個：

- **Project**：一個專案，可包含多個設計（例如不同版本、不同頻率、不同結構）
- **Design**：一個具體的 3D 模型 + 邊界條件 + 求解器設定

你可以把它想成：

- Project = 資料夾
- Design = 一個完整的 EM 模型（含 geometry、ports、mesh、solution）

這個概念很重要，因為封裝互連常常需要：

- 同一個 project 裡放多個 design（不同 via 結構、不同 trace geometry）
- 用 parametric sweep 比較不同尺寸
- 用 Optimetrics 做最佳化

# 2. HFSS 的座標系（這是你畫 3D 結構的基礎）

HFSS 使用右手座標系：

- **X、Y**：平面方向（通常是 PCB 平面）
- **Z**：垂直方向（介質厚度、via 深度）

封裝互連模型中常見的 mapping：

- BGA pad → XY 平面
- Die bump → XY 平面
- Substrate 層 stack-up → Z 方向
- Via → 沿 Z 方向穿透

你後面做 package via、microstrip、stripline、coax transition，全都依賴這個座標系。

---

# 3. Driven Modal vs Driven Terminal（最重要的 solver 選擇）

這是 HFSS 初學者最常搞錯的地方，但對封裝互連至關重要。

## Driven Modal（模態求解）

適用於：

- **微帶線 / 帶狀線**
- **差分對**
- **一般 PCB trace**
- **waveguide-like 結構**

它的特點：

- 求解「模態」：電磁波在結構中能存在的場型
- Port 是「模態 port」
- 適合傳輸線、互連、封裝、connector

你做 package interconnect（BGA → substrate → via → trace）時，**幾乎全部用 Driven Modal**。

---

## Driven Terminal（端子求解）

適用於：

- **多導體傳輸線（multi-conductor）**
- **coaxial cable**
- **connector pin field**
- **需要明確定義電壓 / 電流的端子**

它的特點：

- Port 是「terminal port」
- 可以定義多個 conductor 之間的電壓差
- 適合 coax、cable、connector pin

封裝互連中，只有在：

- 你要 model coax → package transition
- 或 multi-pin connector

才會用 Driven Terminal。

# 4. HFSS 的基本 workflow（你後面做封裝互連必備）

## Step 1：建立 3D 幾何

包含：

- BGA pad
- Substrate stack-up（介質層）
- Via（barrel + pad + antipad）
- Trace（microstrip / stripline）
- Solder mask
- Air box

這些尺寸都會影響 S11/S21。

---

## Step 2：指定材料（Material）

常用：

- FR4（εr ≈ 4.2, tanδ ≈ 0.02）
- BT substrate（封裝用）
- Copper（σ = 5.8e7）
- Solder mask
- Air

材料決定介質損耗、導體損耗 → 直接影響 S21。

---

## Step 3：設定邊界條件（Boundary）

最重要的：

- **Radiation boundary**（外部空間）
- **Perfect E / Perfect H**（必要時）
- **Finite conductivity**（銅損耗）

這些邊界條件決定 Maxwell 方程式如何被求解。

---

## Step 4：設定 Port（最重要）

常見：

- **Wave port（模態 port）**：用在 microstrip、stripline
- **Lumped port**：用在小結構、via、封裝
- **Terminal port**：用在 coax、connector

Port 的定義會直接影響 S11/S21 的正確性。

---

## 4.1 / 4.2 不同線寬的差分線（微帶線 / 帶狀線）

封裝中：

- 間距越小 → 耦合越強 → Zdiff 下降

## 🧩 第 5 章真正的主題：差分過孔的電磁行為 + HFSS 差分 via 建模技巧

差分過孔比單端過孔複雜得多，因為它同時涉及：

- 奇模（odd mode）  
- 偶模（even mode）  
- 差分阻抗 Zdiff  
- 模式轉換（Sdc、Scd）  
- 回流路徑（return path）  
- 耦合（k）  
- stub 共振  

這些都是封裝互連的核心物理。

---

# 5.1 不同回流地孔間距  
### 這一節在教你：
- GND via 越靠近差分 via → 回流路徑越短 → **阻抗更穩定**  
- GND via 越遠 → 回流路徑變差 → **模式轉換（Sdc、Scd）增加**

### 封裝對應  
- ABF substrate 的 GND via fence  
- Interposer 的 shielding via  
- HBM channel 的 return path  

這是封裝差分 via 設計最重要的概念之一。

---

# 5.2 不同信號孔間距  
### 這一節在教你：
- 信號 via 越靠近 → 耦合越強 → Zdiff 下降  
- 信號 via 越遠 → 耦合越弱 → Zdiff 上升  
- 間距太小 → 模式轉換增加（差分 → 共模）

### 封裝對應  
- micro‑bump pitch  
- TSV pitch  
- RDL via pitch  

封裝密度比 PCB 高 10 倍，這節的物理直接決定封裝 SI。

---

# 5.3 不同介質  
### 這一節在教你：
- Dk 越大 → 寄生電容越大 → Zdiff 下降  
- Dk 越小 → Zdiff 上升  

### 封裝對應  
- ABF（Dk ≈ 3–4）  
- SiO₂（Dk ≈ 3.9）  
- Si interposer（Dk ≈ 11.9）  

封裝材料比 PCB 更複雜，這節的物理會直接用在 CoWoS / SoIC。

---

# 5.4 不同反焊盤（anti-pad）大小  
### 這一節在教你：
- anti-pad 越大 → 寄生電容越小 → Zdiff 上升  
- anti-pad 越小 → 寄生電容越大 → Zdiff 下降  

### 封裝對應  
- ABF via anti-pad  
- TSV landing pad  
- micro‑bump pad  

封裝設計常用 anti-pad 來調整差分阻抗。

---

# 5.5 不同層數  
### 這一節在教你：
- via 穿越更多層 → via barrel 更長 → 電感更大  
- 電感越大 → 高頻損耗越大 → Sdd21 變差  

### 封裝對應  
- interposer via 深度  
- ABF via 深度  
- TSV 深度（最重要）  

TSV 的深度通常是 50–100 µm，這節的物理會直接用到。

---

# 5.6 不同殘樁（stub）長度  
### 這是整章最重要的部分之一  
你會看到：

- stub = 未使用的 via 殘留部分  
- stub 會形成 λ/4 共振  
- 造成嚴重反射（Sdd11）與損耗（Sdd21）  
- stub 越長 → 共振頻率越低 → 問題越嚴重  

### 封裝對應  
- TSV stub（非常重要）  
- ABF via stub  
- micro‑via stub  

TSMC、Intel、AMD 都會用：

- back-drill  
- via-in-pad  
- blind/buried via  

來消除 stub。

# 5.7 不同過孔長度  
### 這一節在教你：
- via 越長 → 電感越大 → Zodd 上升 → Zdiff 上升  
- via 越短 → 性能越好  

### 封裝對應  
- CoWoS interposer via  
- SoIC bonding via  
- ABF via  

封裝 via 的長度是 SI 的關鍵瓶頸之一。

# 🧩 6.1 Solution Types（解算類型）  
這一節的核心是：  
**HFSS 有不同的求解器，適用不同的結構與頻率。**

### 6.1.1 設置為 “HFSS”  
這是最常用的 **3D 全波求解器（finite element method, FEM）**。

適用於：

- 傳輸線  
- 過孔  
- 差分線  
- RDL  
- micro‑bump  
- TSV（小模型）

特點：

- 精準  
- 計算量大  
- 適合封裝互連

---

### 6.1.2 設置為 “HFSS with Hybrid and Arrays”  
這是 **混合求解器（Hybrid Solver）**，會把 FEM + MoM + PO 等方法混合使用。

適用於：

- 大型天線陣列  
- 大面積結構  
- 多重重複單元（periodic structure）  
- 大型封裝（interposer 大面積）

特點：

- 計算速度快  
- 記憶體需求低  
- 適合大模型（但精度略低）

---

### 6.1.3 結果對比  
你會看到：

- 小模型 → HFSS（FEM）最準  
- 大模型 → Hybrid 更快，但可能有誤差  
- 封裝互連 → 90% 用 HFSS（FEM）

**封裝模擬工程師必須知道何時用哪一種。**

---

# 🧩 6.2 Solution Frequency（求解頻率）  
這一節在教你：  
**HFSS 的求解頻率設定錯了，整個模擬都會錯。**

### 6.2.1 設為 80 GHz  
你會看到：

- mesh 變得非常細  
- 計算時間暴增  
- 記憶體需求暴增  
- 但如果你的訊號只跑到 10–20 GHz → 完全沒必要

### 6.2.2 設為 10 GHz  
你會看到：

- mesh 較粗  
- 計算快  
- 但如果你的訊號跑到 40–60 GHz → 結果會錯

### 6.2.3 結果對比  
你會學到：

- **求解頻率 = 你要分析的最高頻率 × 1.2–2 倍**  
- **頻率越高 → mesh 越細 → 計算越慢**

對封裝的應用：

- HBM：3–7 GHz  
- PCIe 5/6：16–32 GHz  
- SerDes 112G：30–40 GHz  
- RDL / TSV：可能到 60–80 GHz

# 🧩 6.3 Solution Solver（求解器類型）  
這一節在教你：  
**HFSS 有兩種求解器：Direct Solver 與 Iterative Solver。**

### 6.3.1 Iterative Solver  
特點：

- 記憶體需求低  
- 適合大模型  
- 速度快  
- 精度略低  

適用於：

- 大面積 interposer  
- 大型封裝  
- 多層 ABF substrate  

---

### 6.3.2 Direct Solver  
特點：

- 精度最高  
- 記憶體需求巨大  
- 速度慢  

適用於：

- 小模型（via、bump、TSV）  
- 高精度需求  
- 高頻 (>40 GHz)

不同結構 → 不同 solver

這是 TSMC 封裝模擬工程師最重視的能力。
